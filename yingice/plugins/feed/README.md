# feed

## 插件描述

投喂互动插件。用户 @bot 并输入食物名称后，插件会读取 `feedlist.csv` 判断食物类型，回复对应文案，并调用好感度系统修改好感。

## 指令说明

```text
@bot 投喂 食物名称
```

示例：

```text
@bot 投喂 苹果
```

匹配成功后会回复 CSV 中配置的文案，并追加好感变更结果。

回复会以 @投喂人 开头，并附带固定文本：

```text
@投喂人 冰莹收到了你的投喂！
```

## 其他内容

配置文件位于当前插件目录：

```text
feedlist.csv
```

CSV 列含义：

```text
食物名称（以空格为间隔）,回复语句,最低上涨好感,最高上涨好感,是否为正则匹配
```

- 第一列会按空格拆分为食物名称列表。
- 第三、第四列会生成随机好感变化值，包含上下限。
- 第五列为 `T` 时，输入内容包含任一食物名称即可匹配。
- 第五列为 `F` 时，输入内容必须与某个食物名称完全一致。
- 第五列为 `B` 时，作为最低优先级规则，在所有 `T/F` 规则之后匹配。
- 回复语句中的 `\n` 会转换为换行。

好感变更由 `ying_point_system` 插件提供。

如果输入食物完全无法匹配，会回复：

```text
唔……master不让我吃不认识的东西……
```

同时记录到当前插件目录的 `unknow_food` 文件中，列为：

```text
名称,首个提名人的QQ,提名次数
```
